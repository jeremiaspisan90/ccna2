-also known for manual routing
-applying static routes requires 2 way to configure
recipe: ip route
         <not directly connected Network IP>
         <not directly connected subnet mask>
         <next hop host ip>

DYNAMIC ROUTING (EIGRP, OSPF, BGP)
-It only requires (advertise) directly connected network
-Any discovered routes will do adjacency

EIGRP Routing
-Cisco proprietary
-Most efficient routing protocol

recipe:
router eigrp_(Autonomous System Number)
  network <direckly connected network IP)
  no auto-summary
 end

@D1
config t
 router eigrp 100
   no auto-summary
   network  10.1.4.4
   network  10.2.1.0
   network  10.2.2.0
   network  192.168.1.128


@D2

   network  10.1.4.8
   network  10.2.1.0
   network  10.2.2.0
   network  192.168.1.128

OSPF Routing 
 -Advertised by AREA (Area 0 is the backbone)
 -OSPF States:Down > Init > 2-Way > EXTART -> Echange > Loading > Full
 
 recipe:
   router ospf # (process id)
    router-id (#.#.#.#)
    network (direckly connected NET Ip) (Wildcard mask)
  end

@R4
conf t
router ospf 1
  router-id 4.4.4.4
  network 4.4.4.4 0.0.0.0 area 34
  network 10.1.1.8 0.0.0.3 area 34
 end
  

@R3
conf t
router ospf 1
  router-id 3.3.3.3
  network 3.3.3.3 0.0.0.0 area 34
  network 10.1.1.4 0.0.0.3 area 0
  network 10.1.1.8 0.0.0.3 area 34
 end

@R2
conf t
router ospf 1
  router-id 2.2.2.2
  network 2.2.2.2 0.0.0.0 area 0
  network 10.1.1.0 0.0.0.3 area 12
  network 10.1.1.4 0.0.0.3 area 0
 end

BGP Routing 
-Border Gateway Protocol
-Mostly Used for WAN
-Mostly used by ISP and Large corporation

recipe:
  router bgp <AS No.>
    neighbor <direct IP of neighboring router> remote-as <remote AS No.>
     :
     :
     network <direckly connected NET.IP to neighbor> <subnet mask>
     :
     :
   end

@R1
conf t
 router bgp 1
 bgp log-neighbor-changes
  neighbor 208.8.8.4 remote-as 45
  neighbor 209.9.9.3 remote-as 3
  neighbor 207.7.7.2 remote-as 2
  network 208.8.8.0 mask 255.255.255.0
  network 209.9.9.0 mask 255.255.255.0
  network 207.7.7.0 mask 255.255.255.0
  network 10.1.1.0 mask 255.255.255.252
end

ROUTE REDISTRIBUTION
(by default, different routing protocols cannot get along each other)

@R1 (for BGP + OSPF)
conf t
router ospf 1
  default-information originate always
end

 @R4 (for OSPF +EIGRP)
!pasok EIGRP > OSPF
conf t
router ospf 1
  redistribute eigrp 100 subnets
exit

!pasok OSPF > EIGRP
router eigrp 100
 redistribute ospf 1 metric 1544 2000 255 1 1500
end
